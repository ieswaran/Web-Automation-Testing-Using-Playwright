Git Commands

1. Setup & Initialization

PurposeCommand
Initialize a local Git repogit init
Clone a remote repositorygit clone <repo-url>
Check repo statusgit status
Set user config (global)git config --global user.name "Your Name"
git config --global user.email "you@email.com"

2. Branching & Navigation

PurposeCommand
View all branchesgit branch
Create a new branchgit checkout -b feature/branch-name
Switch to a branchgit checkout branch-name
Delete a local branchgit branch -d branch-name
Fetch remote branchesgit fetch
Track a remote branchgit checkout -t origin/branch-name

3. Stage, Commit, and Push

PurposeCommand
Stage specific file(s)git add filename.ext
Stage all changesgit add .
Commit staged changesgit commit -m "Descriptive message"
Push current branch to remotegit push origin branch-name

4. Pulling and Merging

PurposeCommand
Pull latest changesgit pull origin branch-name
Merge another branch into currentgit merge branch-name
Abort a merge conflictgit merge --abort

5. Stashing (Temporary Save)

PurposeCommand
Save changes temporarilygit stash
View stash listgit stash list
Reapply last stashed changesgit stash apply
Drop a specific stashgit stash drop stash@{n}

6. Conflict Resolution

PurposeCommand
See conflicting filesgit status
Fix conflicts manually(Open files in editor)
Mark as resolvedgit add conflict-file.ext
Commit after fixing conflictsgit commit

7. Cleaning & Resetting

PurposeCommand
Discard local changes in a filegit checkout -- file.ext
Reset working directorygit reset --hard
Remove untracked filesgit clean -fd

8. Inspecting Repo History

PurposeCommand
View commit historygit log
One-line commit loggit log --oneline
Show specific commitgit show <commit-hash>
Compare branchesgit diff branch1..branch2

9. Remote Repo Management

PurposeCommand
Add remote repogit remote add origin <repo-url>
View remotesgit remote -v
Rename remotegit remote rename origin upstream

10. Tagging Releases

PurposeCommand
Create a taggit tag v1.0.0
Push tags to remotegit push origin --tags

Best Practices

  • Always pull before pushing: git pull origin <branch>
  • Write meaningful commit messages
  • Use feature branches for isolation
  • Use .gitignore to exclude unnecessary files
